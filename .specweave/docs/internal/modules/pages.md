## Server Actions (Primary)

Most mutations use Server Actions instead of REST APIs:

```typescript
import { actions } from 'astro:actions';

// Sign in
const result = await actions.auth.lightSignIn({ email, password });

// Create business
const result = await actions.business.create(formData);

// Admin operations
const result = await actions.admin.listings.getAll();
```

## REST APIs (Limited)

- **Public data caching** (CDN caching support)
- **Scheduled/cron jobs** (background tasks)

## Admin Authentication Flow

```
User → POST /_actions/auth.lightSignIn → Sets cookie
     → GET /api/auth/session → Returns user with role
     → /admin checks role → redirect if not admin
```

### Light Sign In (Server Action)

```typescript
// login.astro
const result = await actions.auth.lightSignIn({ email, password });
if (result.success) {
  // Cookie set automatically
  window.location.href = '/admin';
}
```

### Admin API Auth

```typescript
// src/lib/admin-auth.ts
export async function requireAdmin(cookies) {
  const result = await getAuthenticatedUserFromCookies(cookies);
  if ('error' in result) return result;
  if (!['admin', 'super_admin', 'editor'].includes(result.user.role)) {
    return { error: 'FORBIDDEN' };
  }
  return result;
}
```
